# Kaw Kaw — Customer App (Flutter)

Customer-facing app for grocery, pharmacy, food and parcel delivery.
Flutter · Riverpod · GoRouter · Dio · Firebase Auth/FCM · flutter_map (OSM/OSRM/Nominatim).
**Feature-first** architecture; design system lives in `packages/shared_ui`.

> **Requires Flutter 3.27+ / Dart 3.6+.**

## First-time setup

Platform folders (android/ios) are generated by Flutter. From this directory:

```bash
# 1. Generate native platform scaffolding (keeps the committed AndroidManifest.xml)
flutter create --org in.kawkaw --project-name kawkaw_customer --platforms=android,ios .

# 2. Install dependencies
flutter pub get

# 3. Connect Firebase (project kawkaw08) — see docs/customer_app/firebase-setup.md
dart pub global activate flutterfire_cli
flutterfire configure --project=kawkaw08   # regenerates lib/firebase_options.dart

# 4. Run against a backend
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:3000   # Android emulator -> host
```

`API_BASE_URL` defaults to `http://10.0.2.2:3000` (emulator → host machine). For a device,
pass your machine's LAN IP or the deployed `https://api.kawkaw.in`.

## Architecture

```
lib/
├── main.dart / bootstrap.dart / app.dart      # entry, Firebase init, MaterialApp.router
├── firebase_options.dart                       # generated by flutterfire (placeholder committed)
├── core/
│   ├── config/        AppConfig (env via --dart-define)
│   ├── network/       Dio + auth/refresh interceptor, endpoints, error model, pagination
│   ├── storage/       secure token storage
│   ├── router/        GoRouter (auth-guarded) + bottom-nav shell
│   ├── realtime/      Socket.IO service (order/rider events)
│   ├── maps/          Nominatim, OSRM, geolocator services
│   ├── settings/      public settings (delivery fee, tax…)
│   ├── providers/     core Riverpod providers (dio, storage)
│   └── widgets/       shared scaffolds
└── features/<feature>/
    ├── data/          models + repositories
    ├── application/   Riverpod controllers/providers
    └── presentation/  screens + widgets
```

Features: `auth · home · categories · products (catalog/details/search) · cart · checkout ·
addresses · orders (list/details/tracking) · notifications · profile · support · prescriptions`.

## Identity model

Firebase Phone Auth is the **single source of identity**. The app obtains a Firebase **ID
token** (OTP verified on-device) and sends it to `POST /api/v1/auth/firebase`; the backend
verifies it and returns Kaw Kaw JWTs. The backend never sends OTPs itself.
